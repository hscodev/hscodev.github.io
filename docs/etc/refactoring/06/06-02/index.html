6-2 메서드 내용 직접 삽입

메서드 기능이 너무 단순해서 메서드명만 봐도 너무 뻔할 땐 그 메서드의 기능을 호출하는 메서드에 넣어버리고 그 메서드는 삭제하자.

// as-is
int getRating(){
    return (moreTanFiveLateDeliveries()) ? 2 : 1;
}

boolean moreTanFiveLateDeliveries(){
    return _numberOfLateDeliveries > 5;
}
// to-be
int getRating(){
    return (_numberOfLateDeliveries > 5) ? 2 : 1;
}

동기​

메서드명에 모든 기능이 반영될 정도로 메서드 기능이 지나치게 단순할 때 메서드를 없애자. 인다이렉션을 사용하면 해결되지만 불필요한 인다이렉션은 오히려 장애물이다.

  • 메서드 내용 직접 삽입 기법은 잘못 쪼개진 메서드에도 적용할 수 있다. 전부 큰 메서드 하나에 삽입한 후 각각 다시 작은 메서드로 추출할 수 있다. 메서드를 메서드 객체로 전환 기법을 적용하기 전에 이 기법을 실시하면 좋을 때가 많다.

과다한 인다이렉션과 메서드가 다른 메서드에 단순한 위임을 하고 있어서 코드가 복잡할 때에도 메서드 내용 직접 삽입을 실시한다.


방법​

  • 메서드가 재정의되어 있지 않은지 확인

    → 메서드가 하위클래스에 재정의되어 있다면 메서드 내용 직접 삽입을 실시하지 말자.

  • 그 메서드를 호출하는 부분을 모두 찾는다.

  • 각 호출 부분을 메서드 내용으로 교체한다.

  • 테스트

위의 단계는 간단하지만 사용해서는 안되는 케이스도 있다. 재귀, 여러개의 반환 지점, 접근 메서드가 없을 때 다른 객체 안에 직접 삽입하는 방법 등 복잡한 상황이다.
