6장 메서드 정리

  • 거의 모든 문제점은 장황한 메서드로 인해 생긴다.
  • 장황한 메서드에서는 마구 얽힌 복잡한 로직에 정보들이 묻혀버린다.
  • 핵심적인 리팩토링 기법은 코드 뭉치를 별도의 메서드로 빼내는 메서드 추출이다.
  • 그와 반대로 메서드 내용 직접 삽입은 메서드의 내용을 호출하는 메서드에 직접 넣는 기법이다.
  • 메서드로 빼낸 후 제역할을 못하거나 쪼갠 방식을 바꿔야 할 대는 메서드 내용 직접 삽입 기법을 적용한다.
  • 메서드 추출이 힘든 이유는 임시 변수 때문이다. 이것은 임시변수를 메서드 호출로 전환을 실시해서 없어도 되는 임시변수를 전부 제거한다.
  • 임시 변수가 여러 부분에 사용될 때는 임시변수 분리를 먼저 실시하면 임시변수 메서드 호출로 전환 기법이 더 간편해진다.
  • 임시 변수가 너무 얽혀 메서드 호출로 전환할 수 없을때는 메서드 객체로 전환을 실시하자. 새 클래스에서 심각하게 얽혀 있는 메서드도 분리할 수 있다.
  • 매개변수는 값만 대입하지 않으면 임시변수보다 문제가 덜하다. 하지만 대입하는 값이 있을 땐 매개변수로의 값 대입 제거를 실시해야 한다.
  • 알고리즘도 메서드를 잘게 쪼개면 동작원리를 이해하기 쉽다. 알고리즘 전환을 실시할 수 있다.
