6-7 매개변수로의 값 대입 제거

매개변수로 값을 대입하는 코드가 있을 땐 매개변수 대신 임시변수를 사용하게 수정하자.

// as-is
int discount (int inputVal, int quantity, int uearToDate) {
    if (inputVal > 50) inputVal -= 2;
// to-be
int discount (int inputVal, int quantity, int uearToDate) {
    int result = inputVal;
    if (inputVal > 50) result -= 2;

동기​

매개변수로의 값 대입은 매개변수의 참조를 변경한다는 의미이다. 매개변수에서 특정 처리는 상관없지만 참조를 변경하는 것은 절대로 안된다.

void aMethod(Object foo){
    foo.modifyInSomeWay(); // 괜찮다.
    foo = anotherObject; //절대절대 안된다.(고통과 절망을 안겨줄 것이다.)

코드 명료성도 떨어지고 값을 통한 전달과 참조를 통한 전달을 혼동하게 되기 때문이다. 또한 메서드 안의 코드 자체도 혼동된다.

물론 이 규칙은 출력 매개변수가 사용되는 다른 언어에서는 굳이 적용할 필요 없지만, 그런 언어에서도 출력 매개변수는 가능하면 적게 사용하는 것이 좋다.


방법​

  • 매개변수 대신 사용할 임시변수를 선언하자.

  • 매개변수로 값을 대입하는 코드 뒤에 나오는 매개변수 참조를 전부 임시변수로 수정하자.

  • 매개변수로의 값 대입을 임시변수로의 값 대입으로 수정하자.

  • 컴파일과 테스트를 실시하자.

    → 참조를 통한 전달이고 호출한 함수 밖에서 이 변수가 계속 사용되는지 확인하자. 사용된다고 하면 값 반환을 하게하고 두개 이상의 값 반환이 필요하게 되면 객체로 전환하거나 별도의 메서드로 만들 수 있는지 살펴보자.
