6-9 알고리즘 전환

알고리즘을 더 분명한 것으로 교체해야 할 댼 해당 메서드의 내용을 새 알고리즘으로 바꾸자.

// as-is
String foundPerson(String[] people) {
    for (int i = 0; i < people.length; i++){
        if (people[i].equals ("Don")) {
            return "Don";
        }
        if (people[i].equals ("John")) {
            return "John";
        }
        if (people[i].equals ("Kent")) {
            return "Kent";
        }
    }
    return "";
}
// to-be
String foundPerson(String[] people) {
    List candidates = Arrays.asList(new String[] {"Don", "John", "Kent"});
    for (int i = 0; i < people.length; i++) {
        if (candidates.contains(people[i]))
            return people[i];
    }
}

동기​

어떤 기능을 수행하기 위한 비교적 간단한 방법이 있다면 교체해야 한다.

  • 문제에 관해 점점 알아가다가 더 간단한 방법이 있을을 알게됐을 때.
  • 기능을 지원하기 위해 사용하는 라이브러리의 일부 코드가 자신의 코드와 중복될 때 등

길고 복잡한 알고리즘은 수정하기 어려우므로 먼저 메서드를 최대한 잘게 쪼개 간단한 알고리즘으로 교체하면 작업이 수월하다.


방법​

  • 교체할 간결한 알고리즘을 준비한다. 컴파일을 한다.
  • 새 알고리즘을 실행하면서 여러 번의 테스트를 실시한다. 모든 테스트 결과가 같으면 성곤한 것이다.
  • 결과가 다르게 나온다면, 기존 알고리즘으로 테스트와 디버깅을 실시해 비교한다.
