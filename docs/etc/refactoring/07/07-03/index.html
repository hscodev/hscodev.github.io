7-3 클래스 추출

두 클래스가 처리해야 할 기능이 하나의 클래스에 들어 있을 땐 새 클래스를 만들고 기존 클래스의 관련 필드와 메서드를 새 클래스로 옮기자.


동기​

클래스는 시간이 갈수록 방대해지기 마련이고 이해하기도 힘들어진다.


방법​

  • 클래스의 기능 분리 방법을 정한다.

  • 분리한 기능을 넣을 새 클래스를 작성한다.

  • 원본 클래스에서 새 클래스로의 링크를 만든다.

    → 양방향 링크를 만들어야 할 수도 있는데, 필요할 때까진 역방향 링크를 만들지 않는다.

  • 옮길 필드마다 필드 이동을 적용한다.

    → 필드를 하나씩 옮길 때 마다 컴파일과 테스트를 한다.

  • 메서드 이동을 실시하여 옮긴다. 하급 메서드(피호출)부터 시작해서 상급 메서드(호출)에 적용한다.

  • 메서드 이동을 실시할 떄마다 테스트를 실시한다.

  • 각 클래스를 다시 검사해서 인터페이스를 줄인다.

  • 여러 곳에서 클래스에 접근할 수 있게 할지 결정한다. 여러 곳에서 접근할 수 있게 할 경우, 새 클래스를 참조 객체나 변경불가 값 객체로서 공개할지 여부를 결정한다.


예제​

class Person{
    public String getName(){
        return _name;
    }
    
    public String getTelephoneNumber(){
        return ("(" + _officeAreaCode + ") " + _officeNumber);
    }

    public String getOfficeAreaCode(){
        return _officeAreaCode;
    }

    void setOfficeAreaCode(String arg){
        return _officeAreaCode = arg
    }

    public String getOfficeNumber(){
        return _officeNumber;
    }

    void setOfficeNumber(String arg){
        return _officeNumber = arg
    }

    private String _name;
    private String _officeAreaCode;
    private String _officeNumber;
    
}

전화번호 기능을 하나의 클래스로 떼어낼 수 있다. 클래스를 정의한다.

class TelephoneNumber{
}

Person class에 링크를 작성한다.

class Person{
    private TelephoneNumber _officeTelephone = new TelephoneNumber();
}

필드 이동을 실시한다.

class TelephoneNumber{

    public String getAreaCode(){
        return _areaCode;
    }

    void setAreaCode(String arg){
        return _areaCode = arg
    }

    private String _areaCode;

}

class Person{
    
    public String getTelephoneNumber(){
        return ("(" + getOfficeAreaCode() + ") " + _officeNumber);
    }

    public String getOfficeAreaCode(){
        return _officeTelephone.getAreaCode();
    }

    void setOfficeAreaCode(String arg){
        return _officeTelephone.setAreaCode(arg);
    }
}

이후 나머지 필드를 옮기고 TelephoneNumber 클래스에서 메서드 이동을 실시한다.

class Person{
    public String getName(){
        return _name;
    }
    
    public String getTelephoneNumber(){
        return _officeTelephone.getTelephoneNumber();
    }

    private String _name;
    private TelephoneNumber _officeTelephone = new TelephoneNumber();
}

class TelephoneNumber{
    private String _areaCode;
    private String _number;

    public String getTelephoneNumber(){
        return ("(" + _areaCode + ") " + _number);
    }

    public String getAreaCode(){
        return _areaCode;
    }

    void setAreaCode(String arg){
        return _areaCode = arg
    }

    public String getNumber(){
        return _number;
    }

    void setNumber(String arg){
        return _number = arg
    }
}

이후 새 클래스를 클라이언트에 어느 정도 공개할지 결정한다.

  1. 모든 객체가 Telephonenumber 클래스의 어느 부분이든 변경할 수 있게한다.

    이렇게 하면 TelephoneNumber 클래스가 참조 객체가 되므로 값을 참조로 전환을 실시해야 한다. 이때 Person 클래스가 TelephoneNumber 클래스의 접근 지점이 된다.

  2. 어느 주체든 Person 클래스를 거치지 않고는 TelephoneNumber 클래스의 값을 변경하지 못하게 한다.

    TelephoneNumber 클래스를 변경불가로 만들어야 한다.

  3. TelephoneNumber 클래스를 외부로 전달하기 전에 복사한 후 변경불가로 만든다.

    이 방법은 코드를 보는 사람들이 값을 변경할 수 있다는 착각을 불러일으킨다. 게다가 여기저기로 부수히 전달될 경우 클라이언트 간에 왜곡 문제가 발생할 수도 있다.

클래스 추출은 두 결과 클래스에 따로 락을 걸 수 있어서 병렬 실행 프로그램의 생동감을 향상시키는 용도로 흔히 사용되는 기법이다. 두 객체에 락을 걸 필요가 없다면 이 기법을 사용하지 않아도 된다.

이 방법에도 위험성은 있다. 두 객체에 동시에 락을 걸어야 할 경우 트랜젝션과 다른 종류의 공유 락 영역으로 넘어가게 된다. 이러한 해결은 트랜젝션을 관리자를 작성해야 하는데 프로그래머가 하기엔 부적합한 일이다.
