개발자 개인 블로그를 제작해보자.

개발자가 블로그를 시작할 때 접하게 될 선택지들을 공유합니다.


1. 블로그 플랫폼 선택​

📄 항목​

인터넷에 검색해 보자면 블로그로 활용할 수 있는 많은 플랫폼들이 있다.

아래 주소에서는 각 플랫폼별 추천 사용자 유형이나 난이도, 용도 등을 잘 정리해주어서 참고하기 정말 좋았다.

블로그 플랫폼 정리 - 링크

벨로그, 미디엄, 깃허브, 워드프레스

노션, 네이버, 티스토리

네이버, 티스토리, 브런치


📋 요건​

이 차려진 밥상에서 내가 해야 할 것은 나의 목적에 맞는 플랫폼을 선택하는 것이다.

나에게 필요한 요건은 다음과 같다.

  • 쉬운 포스팅
    • 너무 하드하지 않은 커스텀이나 포스트 작성 방식
    • 세팅 및 유지보수에 노력이 너무 크게 들어갈 경우 주객전도 될 수 있다고 생각한다.
  • 커뮤니케이션
    • 특별한 비용 발생 없이 읽어준 사람의 의견을 받을 수 있을 것
    • 단방향 소통보단 양방향을 통해 나의 개선점도 이끌어 낼 수 있으리라.
  • 디자인
    • 원치 않는 광고가 없고, 내용에 집중할 수 있도록 심플할 것
  • 기타
    • 서비스에 완전 의지하지 않을 것. 광고가 붙을거나, 기능 추가에 대한 오류, 블로그를 옮기기 어려운 구성, 생각지 못한 가격정책 등이 발생할 수 있다고 생각한다.
    • 구지 개발자인 것을 티낼 수 있을 것. (🙇‍♂️)

🤔 선택​

개인적으로 꼽는 개발자 블로그 플랫폼은 다음 항목이다.

벨로그, 미디엄, 깃허브, 노션

벨로그는 개발자들을 위한 블로그라고 소개한다. 디자인도 깔끔하니 좋고 개발자들의 소통이 많아보여 내심 끌리기는 했다. 더불어 블로깅을 위한 기능들을 지속적으로 개발할 터이니 혜택을 누리기 좋아보였다. 다만 한 플랫폼에 종속적으로 의지하고 싶지 않아 제외하려 한다.

미디엄의 경우 국제적으로도 널리 공유될 수 있어서 좋다고 생각한다. 다만 커스텀이 불가능하고, 내 개인만의 블로그로 보여지기 어려워 보였다.

노션은 작성하기 편하고, 용도가 무궁무진하다. 여러 용도로 잘 활용하고 있어서 좋은 서비스라고 생각한다. 다만, 다른용도로서 가볍게 잘 쓰고싶은 마음이 있다.


✅ 결정​

결국은 답정너처럼 깃허브를 선택하여 계속 진행해본다.


2. Github Pages에 대해​

사이트 : https://pages.github.com/

깃허브를 통해 블로깅을 하고싶다면, Github Pages라는 서비스를 이용하게 된다.

깃허브 레포지토리에 저장만 하면 알아서 호스팅해줄게~라는 간단한 소개로 모든걸 설명한다.

기본적으로 도메인도 제공해주며, 정적 페이지의 웹 호스팅 서비스를 제공한다.


3. Static Site Generators 선택​

😮 개요​

정적 페이지. 이쁘게 html 파일을 만들어서 한장한장 올려 블로그를 꾸미면 참 좋다. 다만, 배울 시간도, 꾸밀 시간도 많이 들기 때문에 일정한 디자인을 가진 정적 페이지를 찍어내기 위한 정적 페이지 자동 생성기를 찾게된다.


🤔 선택​

Static Site Generators가 정리된 사이트에서 여러 요소들을 검토하여 선택한다.

사이트 : https://github.com/collections/static-site-generators

사람들이 많이 쓰는 jekyll과 hugo, hexo를 사용한다면, 세팅할 때 다른 블로그를 참고하거나 블로그 테마 등을 쉽게 접할 수 있고, 문제에 대한 많은 질답들을 찾아 볼 수 있을 것이다.


⁉️ 예외​

신기하게도 GithubPages의 설정 중 nojekyll이라는 jekyll과 관련된 부분이 있다. GithubPages와 jekyll이 깊은 관계가 있는 것으로 보이며, 선택에 큰 영향을 줄 수 있는 부분인 것 같다. GithubPages에는 내부에 jekyll 이 설치되어있고, 감지하여 변환 및 호스팅을 한다고 한다.

이 블로그에서 각 요소에 대해 자세히 설명해준다 : Github Pages 블로그 소개 링크


✅ 결정​

최종적으로는 쌩뚱맞겠지만 Docusaurus 를 선택하였다.

GithubPages가 jekyll 친화적이라고 생각 할 수도 있지만, 만들어진 정적 페이지만을 업로드하여 호스팅 할 수 있기 때문에 마음에 드는 어느것을 선택해도 무방할 것이다.

Docusaurus

  • 내심 개발자 블로그를 웹서핑 해보던 중에도 facebook에서 만든 Docusaurus 라는 정적 사이트 생성기에 관심 가지게 되었다.

  • 테마에 대해 참고하게 된 블로그 글은 이것이다. - 협력하는 객체들의 공동체

  • ❗중간팁

    • 기술 블로그 웹서핑 중 최 하단에서 사용된 테마나 정적 페이지 생성기의 정보를 볼 수 있다.

  • 하나의 큰 프로젝트에 saurus라는 이름을 붙이는 재치가 좋았다. 🥰

Jekyll

  • Github Pages에서 jekyll 빌드를 자동으로 해주어 사용자는 크게 처리할 요소들이 없다.
  • 블로그 파일을 repo에 저장하면 전적으로 Github Pages에서 빌드되어 볼 수 있지만 시간이 걸린다. 결국은 로컬에서 빠르게 확인하고 싶을 경우 PC에 루비나 jekyll을 위한 세팅을 해야할 수 있다.
  • 무료 테마들이 많지만 개개인들의 작업물인 경우 특정 환경에서는 UI가 깨지기도 하고, 유지보수를 기대하긴 어렵다.
  • 역사가 오래된 만큼 전체적인 스타일이 올드하다. 물론 시대에 따른 멋진 디자인도 나왔을거라 생각하지만 Docusaurus만큼 마음에 드는 구성과 디자인은 찾기 어려웠다.

4. Discussion (댓글) 서비스 선택​

😮 개요​

블로그 글을 작성한다고 해서 댓글 서비스가 자동적으로 붙지는 않는다. 플러그인과 같이 추가적인 영역을 덧붙여 완성이 되는데, 여러 서비스들이 있으며 잘 알아보고 선택해보자.

🤔 선택​

Disqus, utterances

쉽게 한글 블로그에서 소개되는 서비스가 많지는 않아보인다. 선택지가 둘 중 하나로 보여지며, Disqus의 경우 최근 광고를 붙이게도 된다고 한다. 많은 서비스를 비교해보고 싶었지만 결국은 utterances 로 빠른 선택이 되었다.

Disqus

  • 적용 블로그 글이 굉장히 많아 적용하기 어렵지 않다.
  • 개별적으로 서비스에 가입해야한다.
  • facebook, twiter 등 여러 계정을 지원한다.

utterances

  • 오픈소스이다. (마음에 든다면 Github 기여자가 되어보자)
  • Github issue 등록을 사용하는 방식으로 동작한다고 한다. 따라서 가입 없이 Github 계정이 있다면 소통할 수 있다.

방문자들에 대해 여러가지 궁금한 부분들이 있을 것이다.

Google Analytics를 통해 홈페이지에 쉽게 추가하고 방문자 정보를 추적해줄 수 있다.

어느 블로그 글에 접속을 많이 할까?
무엇이 방문자를 오래 머물게 할까?

6. 검색 노출​

내 블로그를 많은 사람들에게 보여주고 싶다면, 검색에 노출해야한다.

자동으로 구글이나 네이버에 노출되지 않으므로 두가지에 대해 설정하도록 하자.

설정 방법도 URL을 제공하면 끝날정도로 매우 간단하다.

7. 마치며. 🙋​

이 정리를 하게 된 계기로는 우리는 자기 PR의 시대에 살며 나에대해 여러 방면으로 어필해야 할 필요가 있어보였다. 많은 직종들에서도 공통적인 내용이며, 개발자들에게서도 크게 영향받는 부분으로 생각되었다. 그 중의 일환으로 블로그를 시작해 보려 하는데 알아봐야할 것이 단계별로 발생하게 되었고 모든게 정리되어있다면 불필요한 과정일 수도 있을거라고 생각이 들었다.

그러면서 결국 이러한 과정들이 나의 단순한 나의 경험으로 끝나지 않고 정리하여 공유하고자 하였다. 이로서 다른 사람의 1분, 1초라도 아껴줄 수 있었다면 나는 만족할 수 있는 부분이라고 생각한다.
